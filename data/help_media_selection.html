This step allows you to mount an SAP HANA update medium via NFS and, optionally, copy it to the local filesystem.

Mount NFS share
With this option enabled, the Wizard will mount the NFS share specified in the next step to each of the nodes running SAP HANA.
NFS share path
Specify the path to the NFS share here. Please use the same format, as with mount.nfs utility, i.e. servername:/path/to/share.
Copy medium locally
Check this option if you wish the contents of the update medium to be copied locally. Please note that, depending on your network, this can take a considerable amount of time.
Copy path
Specify path on the local node where the contents of the update medium should be copied. This path will be created, if it does not exist. Additionally, the filesystem owner will be set to <SID>adm:sapsys.
